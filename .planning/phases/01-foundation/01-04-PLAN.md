<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: Railway deployment and iPad PWA smoke test</name>
  <what-built>
    Complete project scaffold:
    - FastAPI backend with /api/health endpoint and MongoDB connection
    - React/Vite PWA landing screen with JARVIS orb animation
    - Multi-stage Dockerfile that builds and serves both
    - All config via environment variables (no hardcoded secrets)
  </what-built>
  <how-to-verify>
    **Step 1: Set up Railway (if not already done)**
    1. Go to https://railway.app and create a new project
    2. Connect your GitHub/GitLab repository OR use Railway CLI: `npm install -g @railway/cli && railway login && railway init`
    3. Add the Railway MongoDB plugin: Project → Add Plugin → MongoDB
    4. Add environment variable: PORT = 8080 in Railway project settings
    5. Push to main branch: `git push origin main`
    6. Watch the Railway build logs — it should run the Dockerfile two-stage build

    **Step 2: Verify the deployment**
    Once Railway shows "Success" in the deploy logs:
    1. Open the Railway deployment URL in your browser
    2. Visit `{RAILWAY_URL}/api/health` — must return: `{"status": "ok", "mongo": "connected"}`
    3. Visit `{RAILWAY_URL}/` — must show a dark screen (the browser may not render the orb in desktop view, that is okay for now)

    **Step 3: iPad fullscreen PWA test**
    1. Open Safari on the target iPad
    2. Navigate to the Railway deployment URL
    3. Tap the Share button → "Add to Home Screen"
    4. Tap the JARVIS icon on the home screen
    5. Verify:
       - App opens in fullscreen (NO address bar, NO browser chrome)
       - Background is very dark (#0e0e0e)
       - A glowing orb animation is visible in the center (pulsing blue/purple)
       - "TAP TO SPEAK" text is visible below the orb in light gray
       - Status bar at top shows time/battery overlaid (not pushing content down)
       - No scrollbars, no bounce when attempting to scroll
       - Landscape orientation fills the full screen

    **If Railway build fails:**
    - Check Railway build logs for Docker errors
    - Common issues: missing requirements.txt path, npm ci failing, Python import errors
    - Share the error and we will fix it

    **If iPad shows browser chrome (address bar visible):**
    - The apple-mobile-web-app-capable meta tag may not be in index.html
    - Run: `curl -s {RAILWAY_URL}/ | grep apple-mobile-web-app-capable`
    - If missing, this is a bug in Plan 02 Task 2 to fix before declaring Phase 1 complete
  </how-to-verify>
